3.3.1 Exercises (Aesthetic mappings)

  1. What’s gone wrong with this code? Why are the points not blue?
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy, color = "blue"))

  • The color is being specified within aes(), but it is not varying by a variable, but a constant value. Pull it out and it should work nicely:
ggplot(mpg) +
  geom_point(aes(x  =  displ, y = hwy), color= "blue")

  1. Which variables in mpg are categorical? Which variables are continuous? (Hint: type ?mpg to read the documentation for the dataset). How can you see this information when you run mpg?
  • Can look for <chr in the class for each variable.
  • Categorical: manufacturer, model, trans, drv, fl, class
  • Continuous: displ, cty, hwy, cyl
  1. Map a continuous variable to color, size, and shape. How do these aesthetics behave differently for categorical vs. continuous variables?
ggplot(mpg) +
  geom_point(aes(x=displ, y=hwy, color=displ, size=hwy, shape=drv))

  • Shape cannot plot a continuous variable. Color and Size can plot aesthetics continuously.
  1. What happens if you map the same variable to multiple aesthetics?
ggplot(mpg) +
  geom_point(aes(x=hwy, y=cty, size=cty))

  1. What does the stroke aesthetic do? What shapes does it work with? (Hint: use ?geom_point)
ggplot(mpg) +
  geom_point(aes(x=cty, y=hwy), shape=8, stroke=1.5)

  1. What happens if you map an aesthetic to something other than a variable name, like aes(colour = displ < 5)? Note, you’ll also need to specify x and y.
ggplot(mpg) +
  geom_point(aes(x=cty, y=hwy, color=displ < 5))

  • Can create boolean as an aesthetic (example above of coloring by cylinder count cutoff)

  • Colors the border of the shape (link for shape options). It seems to work for all shapes

3.5.1 Exercises (Facets)

  1. What happens if you facet on a continuous variable?
ggplot(mpg) +
  geom_point(aes(x=displ, y=hwy)) +
  facet_wrap(~cty)

  • It goes ahead an facets on all the options. So…that’ll get outta hand with larger datasets.
  1. What do the empty cells in plot with facet_grid(drv ~ cyl) mean? How do they relate to this plot?
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = drv, y = cyl))

# version with facet_grid
ggplot(mpg) +
  geom_point(aes(x=drv, y=cyl)) +
  facet_grid(drv ~ cyl)

  • The empty cells are where there is no value for that combination of the two categorical variables.
  1. What plots does the following code make? What does . do?
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy)) +
  facet_grid(drv ~ .)

ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy)) +
  facet_grid(. ~ cyl)

  • The code above facets just on one variable. The . is a placeholder (can facet_grid by rows and no columns)
  1. Take the first faceted plot in this section:
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy)) + 
  facet_wrap(~ class, nrow = 2)

What are the advantages to using faceting instead of the colour aesthetic? What are the disadvantages? How might the balance change if you had a larger dataset?

  • Faceting can let you see each variable in isolation, with less crowding it makes it easier to see the distirbution of just one variable. It also means it is harder to see the extent of the full dataset. If the dataset gets larger, it could be more useful to facet to break it up and see individual variables over the whole thing.
  1. Read ?facet_wrap. What does nrow do? What does ncol do? What other options control the layout of the individual panels? Why doesn’t facet_grid() have nrow and ncol arguments?
  • facet_wrap splits on one variable, and nrow and ncol specifies the layout of those facets.
  • as.table can lay things out like a plot (highest in top-right if TRUE) or like a table (default TRUE, highest bottom right).
  • facet_grid doesn’t have nrow or ncol since the grid size is dependent on the unique values of the two variables being faceted on.
  1. When using facet_grid() you should usually put the variable with more unique levels in the columns. Why?
ggplot(mpg) +
  geom_point(aes(x=drv, y=class)) +
  facet_grid(drv ~ class)

  • The unique values in the columns will be labeled on the y axis, easier to read, I’d say.

Geometric objects

  • Visual objects use different geoms (like point versus line)
  • Can keep the same aesthetic mapping in aes() but change the geom to get a different visual.

Separating lines in a line plot? Use linetype in a line geom:

ggplot(mpg, aes(x=displ, y=hwy, color=drv)) +
  geom_smooth(aes(linetype=drv)) +
  geom_point()
## `geom_smooth()` using method = 'loess' and formula 'y ~ x'

  • You can manually group things with group aesthetic, but relying on mapping aesthetic to a discrete variable can be easier (creates a legend for you).
